# 소녀전선:잔불 지원

## 플레이어에게 보이는 결과

`소녀전선_잔불.png`를 가져오면 Lucky가 카드 안의 구조를 알아보고 소녀전선 전용 지휘 콘솔을 바로 연다. 별도 AI 컴파일이나 Lua 실행 승인을 요구하지 않는다.

- 지휘관/전술인형 시작
- 전술인형 획득과 제대 편성
- 매일 5명 고용 목록, 하루 1명 계약, 지정 고용과 숙소 정원
- 작전 선택, 결정론 전투, 승리 보상
- 제조와 수복 대기열
- 기지 시설 강화와 MOD 개조
- 칼리나 상점, 장비 구매·장착
- 전술요정 계약·제대 배치
- 모바일 세로 및 가로 화면

`소녀전선 에셋 모듈.charx`도 함께 선택하면 원본 파일은 브라우저 저장소에 보존하고 ZIP 목차만 먼저 읽는다. 필요한 인형·감정·배경 이미지만 화면이 요청할 때 꺼내므로 554MB 파일 전체를 한꺼번에 해제하지 않는다.

## 가져오는 방법

1. `카드 가져오기`를 누른다.
2. `소녀전선_잔불.png`와 `소녀전선 에셋 모듈.charx`를 함께 선택한다.
3. 대용량 파일 연결 안내를 승인한다.
4. 자동으로 열린 `그리폰 지휘부`에서 시작 방식을 고른다.

에셋 모듈 없이 카드만 먼저 가져온 뒤, 에셋 탭의 `에셋 모듈 연결`로 나중에 붙여도 된다.

## 호환 방식과 안전 경계

Lucky는 원본 카드의 20만 자 Lua를 그대로 실행하지 않는다. 카드에서 인형·장비·임무·시설·제조 규칙을 정적으로 읽어 검증된 `genre.gfl` 모듈과 공용 전투·시간·상점 모듈로 다시 구성한다.

따라서 원본 Lua가 사용하던 채팅 수정, DOM 조작, 저수준 API 권한은 열리지 않는다. 자금·자원·전투 결과는 엔진이 먼저 확정하고 AI는 대화와 장면만 연출한다.

이 정적 컴파일은 필요하지만 Lucky가 가져오기와 카드 열기 때 자동으로 수행한다. 모델 API 키나 사용자의 컴파일 승인은 필요 없다. 이미 저장된 소녀전선 카드도 최신 인증 템플릿을 우선 적용하므로 구버전 초안에 머물지 않는다.

## 원본 고용·시설 규칙

원본 카드의 고용 규칙을 그대로 네이티브 상태로 옮긴다.

- 미보유 인형 중 5명을 매일 추첨
- 같은 날 계약은 한 명만 가능
- 일반 계약은 카드의 `<인형명>d` 변수에 기록된 개별 가격 사용
- 저격 고용은 개별 가격에 자금 3,000 추가
- 숙소 레벨별 정원 3 / 6 / 9 / 12 / 15명
- 계약 직후 상태는 `이동 중`, 다음 시간대에 `대기`로 전환

시설은 Lv.1에서 시작해 Lv.5가 최대이며 다음 비용은 직전 비용의 1.5배를 내림한다.

| 시설 | Lv.1→2 | Lv.2→3 | Lv.3→4 | Lv.4→5 |
| --- | ---: | ---: | ---: | ---: |
| 훈련 시설 | 4,000 / 2,000 | 6,000 / 3,000 | 9,000 / 4,500 | 13,500 / 6,750 |
| 방위 시설 | 4,000 / 2,000 | 6,000 / 3,000 | 9,000 / 4,500 | 13,500 / 6,750 |
| 정비 시설 | 4,000 / 2,000 | 6,000 / 3,000 | 9,000 / 4,500 | 13,500 / 6,750 |
| 보급소 | 5,000 / 3,000 | 7,500 / 4,500 | 11,250 / 6,750 | 16,875 / 10,125 |
| 인형 숙소 | 3,000 / 1,000 | 4,500 / 1,500 | 6,750 / 2,250 | 10,125 / 3,375 |

표의 앞 숫자는 자금, 뒤 숫자는 자원이다.

## 회수와 합성의 구분 (2026-07-17)

수치는 발명하지 않는 것이 원칙이다. 실카드 기준:

- **원본에서 회수**: 인형 206/211명의 HP·MP·전투력·기분(카드 `defaultVariables`의 인형별 배열),
  병과·등급, 아이템·장비·임무·시설 기본 비용 테이블, MOD 개조 전투력(MOD_POWER), 지휘관 시작 자금 10,000.
  카드에 배열이 없는 5명은 원본 Lua 자신의 폴백(HP 1000 · MP 1000 · 전투력 500 · 기분 50)을 쓴다.
- **Lucky 규칙으로 합성**: 전투 판정(d20·치명타·전리품 범위), 제조 비용, 시설 강화 레벨 배율.
  합성 목록은 컴파일 결과 `issues`에 남고 카드 열기 시 진단 콘솔에 기록된다 — 조용한 확정은 없다.

콘텐츠 권리·공유 경계는 [CONTENT-POLICY.md](CONTENT-POLICY.md)를 따른다.

```powershell
corepack pnpm --filter @simbot/web canary:gfl -- "C:\freetalk\소녀전선\소녀전선_잔불.png" "C:\freetalk\소녀전선\소녀전선 에셋 모듈.charx"
```

- 카드 내장 에셋 147개
