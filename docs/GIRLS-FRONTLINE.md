# 소녀전선:잔불 지원

## 플레이어에게 보이는 결과

`소녀전선_잔불.png`를 가져오면 Lucky가 카드 안의 구조를 알아보고 소녀전선 전용 지휘 콘솔을 바로 연다. 별도 AI 컴파일이나 Lua 실행 승인을 요구하지 않는다.

- 지휘관/전술인형 시작
- 보유 인형 0명으로 시작, 고용·제조 뒤 제대 편성
- 매일 5명 고용 목록, 하루 1명 계약, 지정 고용과 숙소 정원
- 레드·오렌지/동유럽/화동 전선을 먼저 고른 뒤 세부 지역 선택, 결정론 전투와 승리 보상
- 승리한 임무의 별점으로 쌓이는 지휘관 EXP·Lv 20 진급, 레벨별 일일 작전 상한과 판정 보정
- 제조와 수복 대기열
- 기지 시설 강화와 MOD 개조
- 카리나 보급 상점/장비 상점/보관함·장비 관리의 단계형 진입
- 전술요정 계약·제대 배치
- 지휘관실·작전실·정비실·숙소·기지 외부 이동, 현장/원격 지휘 구분
- 이동 즉시 엔진 위치를 확정하고 같은 결과를 AI 서사와 다음 대화 프롬프트에 전달
- 오전→오후→저녁→밤→심야→새벽 시간 진행과 한 번에 처리하는 `하루 마감`
- Choice 캡슐 관계 행동과 Dice d20 판정, 판정 결과에 따른 실제 호감도·기분 변화
- 관계 단계(카드 회수 REL_NAMES/REL_THRES)에 따라 해금되는 선택지 사다리 9종, 성공 직후의 후속 캡슐(DC 보정), 같은 행동 반복 시 효과 절반과 하루 교류 상한
- 시간을 멈추는 1:1 대화 세션 — 자유 대화 후 마무리 시 엔진이 교감 보너스를 확정(인형별 하루 1회)
- 원본 상품 효과의 실제 사용·판매, 장비 장착·해제, 제대 슬롯 인형 프로필
- 원본 감정 대사틀·상태창 신호를 Lucky 네이티브 인용 대사와 지휘 상태표로 표시
- 모바일 세로 및 가로 화면

업데이트판의 `v0.5fix 소녀전선 에셋 모듈 1번.charx`부터 `3번.charx`까지 함께 선택하면 원본 파일 3개는 브라우저 저장소에 각각 보존하고 ZIP 목차만 먼저 읽는다. 필요한 인형·감정·배경 이미지만 실제 화면이 요청할 때 해당 모듈에서 꺼내므로 대용량 파일 전체를 한꺼번에 해제하거나 미리 읽지 않는다.

## 가져오는 방법

1. `카드 가져오기`를 누른다.
2. `소녀전선_잔불.png`와 에셋 모듈 1·2·3번을 한꺼번에 선택한다.
3. 세 모듈의 합계 용량이 표시된 대용량 연결 안내를 한 번 승인한다.
4. 자동으로 열린 `그리폰 지휘부`에서 시작 방식을 고른다.

에셋 모듈 없이 카드만 먼저 가져온 뒤, 에셋 탭의 `에셋 모듈 연결`에서 1·2·3번을 함께 골라 나중에 붙여도 된다. 모듈 수에는 1개 제한이 없고, 같은 모듈을 다시 선택해도 중복 연결되지 않는다.

## 호환 방식과 안전 경계

Lucky는 원본 카드의 20만 자 Lua를 그대로 실행하지 않는다. 카드에서 인형·장비·임무·시설·제조 규칙을 정적으로 읽어 검증된 `genre.gfl` 모듈과 공용 전투·시간·상점 모듈로 다시 구성한다.

따라서 원본 Lua가 사용하던 채팅 수정, DOM 조작, 저수준 API 권한은 열리지 않는다. 자금·자원·전투 결과는 엔진이 먼저 확정하고 AI는 대화와 장면만 연출한다.

원본의 45개 표시 정규식과 고정 위치 HTML/CSS는 인증판에서 그대로 실행하지 않는다. 모바일 화면을 덮고 버튼은 작동하지 않는 껍데기 대신 Lucky 네이티브 지휘 콘솔을 사용한다. `[|<img="M1918_joy">|"대사"|]`는 안전한 감정 이미지와 인용 대사로 바꾸고, `[상태창][사이드패널][하단상태창]`은 중복 HTML 대신 엔진이 소유한 지휘 상태표로 대응한다.

AI가 출력한 `[[aff=M1918=5]]`, `[[mood=M1918=25]]`는 그대로 적용하지 않는다. 관계 선택지를 누르는 순간 엔진이 d20을 굴려 성공·실패·대성공·대실패를 확정하고, 그 단계에 허용된 증감만 먼저 반영한 뒤 AI가 장면을 쓴다. AI 제안 태그가 있으면 본문에서 제거하고 적용되지 않았다는 안내를 남긴다.

관계 선택지 해금은 카드 관계표의 절대 배열 번호가 아니라 **호감도 0이 속한 중립 티어로부터의 거리**로
계산한다. 실카드의 적대·경계·불편 같은 음수 티어에서는 달래기 경로인 `차분히 대화한다`만 가능하고,
첫 만남부터 별명·격려가 열린다. 같은 문턱이 연속된 `서약(400)`과 `???(400)`은 첫 항목만 도달 가능한
티어로 해석하므로 호감도 400의 표시와 후속 규칙은 `서약`에 머문다. 회수한 이름과 문턱 값 자체는 바꾸지 않는다.

## 다단계 작전과 전리품

작전은 이제 한 번의 전투로 끝나지 않고 별점과 임무 유형에 따라 2~11단계로 이어진다. 출격 전에 정찰·소탕·섬멸 중 하나를 고르면 전체 단계가 즉시 확정되어 저장되므로, 다시 불러와 유리한 순서로 바꿀 수 없다. 화면의 `⚔/👑/🔍/🚩/❓` 트래커에서 현재 위치와 완료 단계를 확인할 수 있다.

- 정찰은 전투 없이 진행하며 다음 교전 한 번에 명중 +1을 준다.
- 돌발 상황은 자원 발견 여부를 판정한다.
- 미확인 상황은 루팅(40%)·무소속 인형 조우(25%)·무발견(35%)으로 갈린다. 어느 결과든 사건과 대상 선정 RNG를 두 번 소비하며, 루팅이면 이어서 아이템별 드롭 확률을 각각 판정한다.
- 무소속 인형은 원본 제한 풀 145명 중 ban·이미 보유한 인형을 제외하고 고른다. 숙소에 자리가 있으면 무료로 즉시 합류하며 작전당 한 명까지만 영입할 수 있다. 영입 뒤 퇴각해도 인형은 남는다.
- 보스 임무를 처음 클리어하면 26명 중 Gebbennu를 제외한 25명에게 구속·영입 결정이 열린다. 영입 보스는 카드 원본 HP·전투력 그대로 6성 `BOSS` 병과가 되며 일반 제대 전투·관계·장비·수복은 가능하지만 MOD와 제조 출현은 막는다.
- 장비 제조는 원본 일반 풀 22종과 중형 풀 29종을 따로 사용한다. 추첨은 시작할 때 한 번만 소비해 결과를 봉인한다.
- 지휘관실 기록실은 원본 세계관 문서 38건을 본문 그대로 보여준다. 첫 문서는 즉시 열리고 이후 완료 작전 2회마다 순서대로 한 장씩 해금된다.
- 하루 마감 때 방위 시설 레벨에 따라 15%에서 3% 확률로 습격이 발생한다. 제1제대가 자동 방어하며 성공하면 자원 100, 실패하면 자원 10%(최대 500)와 제1제대 HP 15%를 잃는다. 발생하지 않은 날도 같은 두 번의 RNG를 소비한다.
- 인형 해체는 원본 규칙대로 코어 1개와 자원 500~2,000을 회수하며, 장비 해체는 원본 환급 규칙이 없어 가격의 30%를 부품으로 돌려주는 Lucky 규칙을 쓴다. 둘 다 확인 대화상자를 거쳐야 한다.
- 교전에서 입은 인형 HP와 얻은 전리품은 다음 단계로 이어진다. 단계 사이 퇴각하면 전리품은 지키지만 임무 완료 보상과 지휘 EXP는 받지 못한다.
- 아이템 19종의 `drop%`는 원본 `ITEM_DATA`에서 회수했다. 장비 `EQUIP_DATA`에는 드롭 값이 없으므로 장비는 작전 전리품에 포함하지 않는다.

빠른 교전은 `자동 진행`으로 정지 지점(무소속 인형 발견·포로·패배·보스 직전·완료)까지 여러 단계를 LLM 호출 1회로
연속 해소한다. 단계마다 장면을 보고 싶으면 고급 설정의 `작전 서사 · 단계별`을 켠다. 전술 교전의 전술 3택과
지휘 개입은 채팅 결정 카드에서도 바로 고를 수 있고, 관리 행동 후 관리 화면은 기본적으로 유지된다(설정에서 끔 가능).

각 단계의 장면 지침도 원본 `EV_GUIDE`를 그대로 사용한다. 예를 들어 정찰 단계에서는 AI가 임의로 교전을 만들지 못하며, 실제 성공·실패·드롭 수치는 항상 엔진 로그를 따른다.

## 장면 BGM

2026-07-18 제품 오너 결정으로 장면 BGM 기능을 제거했다. 원본 카드와 에셋 모듈에는 음원이 없고 별도 Risu 플러그인이 외부 YouTube 영상을 재생하는 구조여서 Lucky의 안정적인 네이티브 기능으로 유지하기 어렵다고 판단했다.

플레이 화면에는 BGM 바와 재생 조작을 표시하지 않으며 모델에도 BGM 태그 출력을 요구하지 않는다. 과거 응답에 저장된 `|BGM_...|` 및 구형 BGM 헤더는 채팅 본문에서만 조용히 숨긴다.

이 정적 컴파일은 필요하지만 Lucky가 가져오기와 카드 열기 때 자동으로 수행한다. 모델 API 키나 사용자의 컴파일 승인은 필요 없다. 이미 저장된 소녀전선 카드도 최신 인증 템플릿을 우선 적용하므로 구버전 초안에 머물지 않는다.

## 원본 고용·시설 규칙

원본 카드의 고용 규칙을 그대로 네이티브 상태로 옮긴다.

- 시작 보유 인형은 0명이며 M4A1을 자동 지급하지 않음
- 미보유 인형 중 5명을 매일 추첨
- 같은 날 계약은 한 명만 가능
- 일반 계약은 카드의 `<인형명>d` 변수에 기록된 개별 가격 사용
- 저격 고용은 개별 가격에 자금 3,000 추가
- 숙소 레벨별 정원 4 / 8 / 12 / 16 / 20명
- 계약 직후 상태는 `이동 중`, 다음 시간대에 `대기`로 전환

시설은 Lv.1에서 시작해 Lv.5가 최대이며 다음 비용은 직전 비용의 1.5배를 내림한다.

| 시설 | Lv.1→2 | Lv.2→3 | Lv.3→4 | Lv.4→5 |
| --- | ---: | ---: | ---: | ---: |
| 훈련 시설 | 4,000 / 2,000 | 6,000 / 3,000 | 9,000 / 4,500 | 13,500 / 6,750 |
| 방위 시설 | 4,000 / 2,000 | 6,000 / 3,000 | 9,000 / 4,500 | 13,500 / 6,750 |
| 정비 시설 | 4,000 / 2,000 | 6,000 / 3,000 | 9,000 / 4,500 | 13,500 / 6,750 |
| 보급소 | 5,000 / 3,000 | 7,500 / 4,500 | 11,250 / 6,750 | 16,875 / 10,125 |
| 인형 숙소 | 3,000 / 1,000 | 4,500 / 1,500 | 6,750 / 2,250 | 10,125 / 3,375 |

표의 앞 숫자는 자금, 뒤 숫자는 자원이다.

시설 설명은 장식 문구가 아니라 실제 계산에 연결된다. 훈련 시설은 제대 작전능력 +10/15/20/30/50%, 방위 시설은 일일 습격 확률 감소와 방어 작전능력 보너스, 정비 시설은 인형 최대 HP +50/100/150/200/300, 보급소는 하루 자원 +300/600/1000/1500/2500, 숙소는 위 정원과 일일 수면 회복을 적용한다. 원문이 `수면 회복량+`처럼 등급만 적은 부분의 회복률은 Lucky 규칙(10/20/30/40/100%)이며 진단에서 합성값으로 밝힌다.

## Choice·Dice 편입

제작자 허가가 기록된 `⚖️ Choice Module: Capsule Extension v5.9`와 `🎲 Dice Module`의 실행형 Lua/DOM 조작은 가져오지 않고 메커니즘만 엔진 방식으로 재구현한다. 전투는 기존 d20 명중·치명타 판정을 계속 쓰고, 관계 캡슐은 현재 호감도와 기분에서 보정치를 계산한다. 사용자가 DC·주사위·증감 숫자를 직접 주입할 수 없고 모든 굴림은 세이브와 롤백에 포함된다.

## 회수와 합성의 구분 (2026-07-17)

수치는 발명하지 않는 것이 원칙이다. 실카드 기준:

- **원본에서 회수**: 인형 206/211명의 HP·MP·전투력·기분(카드 `defaultVariables`의 인형별 배열),
  병과·등급, 6칸 3열 제대 구조, 임무 별점(`diff`의 ★ 개수)·적 문자열의 세력(철혈·바랴그단·E.L.I.D·패러데우스)과 보스,
  아이템·장비·임무·시설 기본 비용 테이블, 아이템 19종의 drop%, 별점별 단계 수(PROG_BY_STAR), 임무 유형 3종(MISSION_TYPES),
  단계 서사 지침 5종(EV_GUIDE), MOD 개조 전투력(MOD_POWER), 지휘관 시작 자금 10,000.
  무소속 인형 조우 풀 145명과 빈 ban 목록, 이미 보유한 인형 제외·무료 즉시 합류·빈 보유 슬롯 검사도 원본에서 회수했다.

## 2026-07-19 연애 리듬

- 인형은 병과에 따른 선호·비선호 관계 행동을 가지며 선호 성공 전까지 상세 화면에는 `?`만 보인다. 공개 뒤 선호 캡슐에 ♥가 붙고, 선호/비선호는 DC −2/+2를 적용한다.
- 계약일로부터 7·30·100일째에는 복수 인형의 기념일 카드가 동시에 열리고, 그날 해당 인형의 관계 행동 효과가 1.5배가 된다. 신뢰 이상 인형과의 외출은 판정 없이 호감·기분을 올리는 대신 실제 시간대 한 칸을 소비한다.
- 대파·패닉 후보는 전투 맥락에 따라 야간/포화/철혈 공포 하나로 확정된다. 1:1 대화 중 심리 치유에 세 번 성공하면 공포를 없애고 같은 조건 명중 +1인 극복 훈장을 남긴다.
- 대화 마무리에서 엔진이 작전 동행·빠른 수복·기념일 약속을 요청할 수 있다. 수락 약속은 최대 3개이며 이행과 위반 모두 하루 마감 영수증으로 확정된다.
- 질투는 끔/순한맛/진한맛을 고를 수 있다. 모든 모드가 같은 d100을 소비하고, 실제 난입은 같은 인형이 연속 표적이 되지 않도록 막는다.
- 비밀 취미는 인형별 결정론 태그로 저장하되 발견 전에는 어떤 화면에도 노출하지 않는다. 관계 대성공 뒤 호칭은 모델이 아니라 사용자가 20자 안에서 한 번 직접 정하며 이후 관계 판정 +1을 받는다.
